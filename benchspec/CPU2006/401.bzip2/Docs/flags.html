401.bzip2 Flags

These are flags that appear in the benchmark sources, categorized as to whether they are portability candidates, mandatory, or forbidden.

Sections

Selecting one of the following will take you directly to that section:

Portability Flags

  • -DBZ_EXPORT
  • -DBZ_LCCWIN32
  • -DBZ_UNIX
  • -DMSDOS
  • -DOS2
  • -D_WIN32
  • -D__CYGWIN__
  • -D__DJGPP__
  • -D__GNUC__
  • -D__cplusplus
  • SPEC_NULLCAST
    • -DBZ_EXPORT
    • [-/]DBZ_EXPORT(?:=\S*)?\b

    • This flag indicates that the functions from bzlib should not be imported from a DLL. This flag is set by default in the benchmark.

    • -DBZ_LCCWIN32
    • [-/]DBZ_LCCWIN32(?:=\S*)?\b

    • This flag indicates that the benchmark is being built with lcc under Windows.

    • -DBZ_UNIX
    • [-/]DBZ_UNIX(?:=\S*)?\b

    • In the original bzip2 source, this flag indicates that the source is being built on a Unix-like system. As this primarily concerns file I/O (which the benchmark does not do), all systems get this flag in the benchmark version.

    • -DMSDOS
    • [-/]DMSDOS(?:=\S*)?\b

    • This flag indicates that the source is being built on MS-DOS. While that seems unlikely, it's not forbidden.

    • -DOS2
    • [-/]DOS2(?:=\S*)?\b

    • This flag indicates that the source is being built on OS/2. While that seems unlikely, it's not forbidden.

    • -D_WIN32
    • [-/]D_WIN32(?:=\S*)?\b

    • This flag indicates that the source is being built on Windows.

    • -D__CYGWIN__
    • [-/]D__CYGWIN__(?:=\S*)?\b

    • This flag indicates that the source is being built under Windows using the Cygwin toolset.

    • -D__DJGPP__
    • [-/]D__DJGPP__(?:=\S*)?\b

    • This flag indicates that the source is being built under MS-DOS using the DJGPP compiler.

    • -D__GNUC__
    • [-/]D__GNUC__(?:=\S*)?\b

    • This flag indicates that the source is being built with the GNU Compiler Collection. It has no effect in the CPU2006 version of the code.

    • -D__cplusplus
    • [-/]D__cplusplus(?:=\S*)?\b

    • This flag indicates that the source is being built with a C++ compiler.

    • SPEC_NULLCAST
    • [-/]DSPEC_NULLCAST(?:=\S*)?

    • This macro defines a cast which is used when comparing a file handle value (from the I/O abstraction layer) to NULL. It must match the type used for the file handle, which is int.

Mandatory Flags

  • -DSPEC_BZIP
  • -DSPEC_CPU
    • -DBZ_UNIX
  • -DSPEC_CPU_WINDOWS
    • -DSPEC_BZIP
    • [-/]DSPEC_BZIP(?:=\S*)?\b

    • This flag indicates that the benchmark should be built to run in the CPU2006 harness. Without the flag, there is no hope that the benchmark will work.

    • -DSPEC_CPU
    • [-/]DSPEC_CPU(?:=\S*)?\b

    • This flag is set by the CPU2006 tools harness when building benchmark sources.

    • Includes:
      • -DBZ_UNIX
    • -DSPEC_CPU_WINDOWS
    • [-/]DSPEC_CPU_WINDOWS(?:=\S*)?\b

    • This flag indicates that the benchmark is being built on Windows.

Forbidden Flags

  • -DBZ_DEBUG
  • BZ_EXTERN
  • BZ_API
  • -DBZ_IMPORT
  • -DBZ_NO_STDIO
  • -DBZ_STRICT_ANSI
  • -DDEBUG
  • -DDEBUG_DUMP
  • -DNULL
  • -DTIMING_OUTPUT
  • -D_BZLIB_H
  • -D_BZLIB_PRIVATE_H
  • -Dsmall
  • SPEC_STDIN
  • SPEC_STDOUT
  • SPEC_STDERR
    • -DBZ_DEBUG
    • [-/]DBZ_DEBUG(?:=\S*)?\b

    • Use of this flag enables some output which should never be generated.

    • BZ_EXTERN
    • [-/]DBZ_EXTERN(?:=\S*)?

    • This macro defines text that should be used to indicate that a function is externally defined (as in a dynamic library). Because the benchmark does not build or use dynamic libraries, setting this flag to something other than the default is forbidden.

    • BZ_API
    • [-/]DBZ_API(?:\([^\)]+\)=\S*)?

    • This macro defines text that should be used to indicate how a function should be marked in a Windows library or DLL. Because the benchmark does not build a library, setting this flag to something other than the default is forbidden.

    • -DBZ_IMPORT
    • [-/]DBZ_IMPORT(?:=\S*)?\b

    • Use of this flag indicates that the bzlib functions should be imported from the DLL. Because the benchmark does not build or use DLL, use of this flag is forbidden.

    • -DBZ_NO_STDIO
    • [-/]DBZ_NO_STDIO(?:=\S*)?\b

    • Use of this flag indicates that stdio should not be used. Because it also cuts out a lot of the file I/O functions, use of this flag is forbidden.

    • -DBZ_STRICT_ANSI
    • [-/]DBZ_STRICT_ANSI(?:=\S*)?\b

    • This flag has no effect in 401.bzip2.

    • -DDEBUG
    • [-/]DDEBUG(?:=\S*)?\b

    • This flag enables copious debug output which is guaranteed to cause the benchmark to miscompare. Its use in reportable runs is forbidden.

    • -DDEBUG_DUMP
    • [-/]DDEBUG_DUMP(?:=\S*)?\b

    • This flag causes buffers generated during the benchmark run to be dumped to files. Because this causes actual file I/O, its use in reportable runs is forbidden.

    • -DNULL
    • [-/]DNULL(?:=\S*)?\b

    • NULL should be defined by your system headers. Setting it explicitly is forbidden.

    • -DTIMING_OUTPUT
    • [-/]DTIMING_OUTPUT(?:=\S*)?\b

    • Use of this flag causes some internal timing to be done and output via the debug output facility. Its use is forbidden for reportable runs.

    • -D_BZLIB_H
    • [-/]D_BZLIB_H(?:=\S*)?\b

    • This macro is a guard against multiple inclusion of bzlib.h. Setting it explicitly is forbidden.

    • -D_BZLIB_PRIVATE_H
    • [-/]D_BZLIB_PRIVATE_H(?:=\S*)?\b

    • This macro is a guard against multiple inclusion of bzlib_private.h. Setting it explicitly is forbidden.

    • -Dsmall
    • [-/]Dsmall(?:=\S*)?

    • This flag is sometimes implicitly set on Windows; in the bzip2 source code, if it is present, it is undefined.

    • SPEC_STDIN
    • [-/]DSPEC_STDIN(?:=\d*)?

    • This flag is used to set the file number for stdin used by the benchmark's file I/O abstraction layer. Its default value must not be changed.

    • SPEC_STDOUT
    • [-/]DSPEC_STDOUT(?:=\d*)?

    • This flag is used to set the file number for stdout used by the benchmark's file I/O abstraction layer. Its default value must not be changed.

    • SPEC_STDERR
    • [-/]DSPEC_STDERR(?:=\d*)?

    • This flag is used to set the file number for stderr used by the benchmark's file I/O abstraction layer. Its default value must not be changed.
