458.sjeng Flags

These are flags that appear in the benchmark sources, categorized as to whether they are portability candidates, mandatory, or forbidden.

  • Forbidden Flags

Forbidden Flags

  • -DFTIME
  • -DGETTICKCOUNT
  • -DGETTIMEOFDAY
  • -DINPROBECODE
  • -DPN2
  • -DPROTOS_H
  • -DSJENG_H
  • -Dmax
  • -Dmin
    • -DFTIME
    • [-/]DFTIME(?:=\S*)?\b

    • This flag is forbidden because in the context of SPEC CPU, time is measured by the SPEC harness, rather than by the benchmark itself.

    • -DGETTICKCOUNT
    • [-/]DGETTICKCOUNT(?:=\S*)?\b

    • This flag is forbidden because in the context of SPEC CPU, time is measured by the SPEC harness, rather than by the benchmark itself.

    • -DGETTIMEOFDAY
    • [-/]DGETTIMEOFDAY(?:=\S*)?\b

    • This flag is forbidden because in the context of SPEC CPU, time is measured by the SPEC harness, rather than by the benchmark itself.

    • -DINPROBECODE
    • [-/]DINPROBECODE(?:=\S*)?\b

    • It seems likely that setting this flag would break the build; in any case, it's untested, and forbidden in a SPEC run.

    • -DPN2
    • [-/]DPN2(?:=\S*)?\b

    • If set, this would cause sjeng to search by a quite different method than the one used in the SPEC benchmark.

    • -DPROTOS_H
    • [-/]DPROTOS_H(?:=\S*)?\b

    • This flag is used to prohibit multiple includes of a header file. Explicitly defining this flag would presumably break the build; setting this flag on the command-line during compile is therefore forbidden.

    • -DSJENG_H
    • [-/]DSJENG_H(?:=\S*)?\b

    • This flag is used to prohibit multiple includes of a header file. Explicitly defining this flag would presumably break the build; setting this flag on the command-line during compile is therefore forbidden.

    • -Dmax
    • [-/]Dmax(?:=\S*)?\b

    • max and min are macros defined in sjeng.h; defining them on the command line would presumabbly break the build, so they are forbidden.

    • -Dmin
    • [-/]Dmin(?:=\S*)?\b

    • max and min are macros defined in sjeng.h; defining them on the command line would presumabbly break the build, so they are forbidden.
