481.wrf Flags

These are flags that appear in the benchmark sources, categorized as to whether they are portability candidates, mandatory, or forbidden.

    • -DALLOCA_ARG_T
    • [-/]DALLOCA_ARG_T(?:=\S*)?\b

    • This macro defines the type of argument that the alloca() function takes. It defaults to 'int'.

    • -DNOUNDERSCORE
    • [-/]DNOUNDERSCORE(?:=\S*)?\b

    • This macro indicates that C functions called from Fortran should not have an underscore added to their names.

    • -DF2CSTYLE
    • [-/]DF2CSTYLE(?:=\S*)?\b

    • This flag should be used if C function names need to have two underscores appended in order for them to be callable by Fortran functions.

    • -DSPEC_CPU_P64
    • [-/]DSPEC_CPU_P64(?:=\S*)?\b

    • This macro indicates that the benchmark is being compiled on a system where integers and longs are 32 bits wide, and pointers are 64 bits wide.

    • -DSPEC_CPU_LP64
    • [-/]DSPEC_CPU_LP64(?:=\S*)?\b

    • This macro indicates that the benchmark is being compiled on a system where integers are 32 bits wide, and longs and pointers are 64 bits wide.

    • -DSPEC_CPU_ILP64
    • [-/]DSPEC_CPU_ILP64(?:=\S*)?\b

    • This macro indicates that the benchmark is being compiled on a system where integers, pointers, and longs are all 64 bits wide.

    • -DSPEC_CPU_LINUX
    • [-/]DSPEC_CPU_LINUX(?:=\S*)?\b

    • This macro indicates that the benchmark is being compiled on a Linux system.

    • -DSPEC_CPU_CASE_FLAG
    • [-/]DSPEC_CPU_CASE_FLAG(?:=\S*)?\b

    • This macro indicates that Fortran functions called from C should have their names lower-cased.

    • -DSPEC_CPU_LINUX64_IPF
    • [-/]DSPEC_CPU_LINUX64_IPF(?:=\S*)?\b

    • This macro indicates that the benchmark is being compiled on an Itanium Processor Family-based Linux system.

    • -DSPEC_CPU_LINUX_PPC
    • [-/]DSPEC_CPU_LINUX_PPC(?:=\S*)?\b

    • This macro indicates that the benchmark is being compiled on a PowerPC-based Linux system.

    • -DSPEC_CPU_LINUX_SPARC
    • [-/]DSPEC_CPU_LINUX_SPARC(?:=\S*)?\b

    • This macro indicates that the benchmark is being compiled on a SPARC-based Linux system.

    • -DSPEC_CPU_MACOSX_INTEL
    • [-/]DSPEC_CPU_MACOSX_INTEL(?:=\S*)?\b

    • This macro indicates that the benchmark is being compiled on an Intel-architecture MacOS X system.

    • -DSPEC_CPU_MACOSX
    • [-/]DSPEC_CPU_MACOSX(?:=\S*)?\b

    • This macro indicates that the benchmark is being compiled on a MacOS X system.

    • -DSPEC_CPU_AIX
    • [-/]DSPEC_CPU_AIX(?:=\S*)?\b

    • This macro indicates that the benchmark is being compiled on a PowerPC-based AIX system.

    • -DSPEC_CPU_SOLARIS
    • [-/]DSPEC_CPU_SOLARIS(?:=\S*)?\b

    • This macro indicates that the benchmark is being compiled on a SPARC-based Solaris system.

    • -DSPEC_CPU_SOLARIS_X86
    • [-/]DSPEC_CPU_SOLARIS_X86(?:=\S*)?\b

    • This macro indicates that the benchmark is being compiled on an x64- or x86-based Solaris system.

    • -DSPEC_CPU_WINDOWS_ICL
    • [-/]DSPEC_CPU_WINDOWS_ICL(?:=\S*)?\b

    • This macro indicates that the benchmark is being built on a Windows system using the Intel C++ compiler.

    • -DSPEC_CPU_NO_HAVE_ALLOCA
    • [-/]DSPEC_CPU_NO_HAVE_ALLOCA(?:=\S*)?\b

    • This macro indicates that alloca is not available as a macro or function.

    • -DSPEC_CPU_NO_HAVE_ALLOCA_H
    • [-/]DSPEC_CPU_NO_HAVE_ALLOCA_H(?:=\S*)?\b

    • This macro indicates that the alloca.h header file is not present and should not be included.

    • -DSPEC_CPU_NO_NCCONFIG_SOLARIS
    • [-/]DSPEC_CPU_NO_NCCONFIG_SOLARIS(?:=\S*)?\b

    • When set, this macro will inhibit inclusion of the Solaris-specific ncconfig header file.

    • -DSPEC_CPU_NO_FILE_OFFSET_BITS
    • [-/]DSPEC_CPU_NO_FILE_OFFSET_BITS(?:=\S*)?\b

    • If the _FILE_OFFSET_BITS macro is not set, this flag indicates that a default value of "64" should not be provided.

    • -DSPEC_CPU_NO_HAVE_FTRUNCATE
    • [-/]DSPEC_CPU_NO_HAVE_FTRUNCATE(?:=\S*)?\b

    • When set, this macro indicates that the ftruncate function is not available.

    • -DSPEC_CPU_NO_STDC_HEADERS
    • [-/]DSPEC_CPU_NO_STDC_HEADERS(?:=\S*)?\b

    • This flag may be used to keep the macro STDC_HEADERS from being defined by default.

    • -DSPEC_CPU_NO_ST_BLKSIZE
    • [-/]DSPEC_CPU_NO_ST_BLKSIZE(?:=\S*)?\b

    • This macro indicates that "struct stat" does not contain a member called st_blksize.

    • -DSPEC_CPU_WORDS_LITTLEENDIAN
    • [-/]DSPEC_CPU_WORDS_LITTLEENDIAN(?:=\S*)?\b

    • This macro indicates that the system building the benchmark is a little-endian system.

    • -DSPEC_CPU_UCHAR_DEFINED
    • [-/]DSPEC_CPU_UCHAR_DEFINED(?:=\S*)?\b

    • If set, this macro indicates that a symbol named "uchar" is already defined to be "unsigned char".

    • -DLOGICAL_STRICT, -DSPEC_CPU_LOGICAL_STRICT
    • [-/]D(?:SPEC_CPU_)?LOGICAL_STRICT\b

    • LOGICAL_STRICT is a NetCDF macro that should be defined for Fortran compilers that treat .eqv./.neqv. as .eq./.ne..

Other Flags

  • -Dalloca
    • -Dalloca
    • [-/]Dalloca(?:=\S*)?\b

    • This allows alloca to be set to the compiler's preferred alloca by SPEC rules.

Mandatory Flags

  • -w
  • -m literal.pm
  • -I., -I./netcdf/include

  • -DTRIEDNTRUE

  • -DNMM_MAX_DIM

  • -DINTIO
  • -DNETCDF
  • -DNO_NAMELIST_PRINT
    • -w
    • specpp
    • -w\b

    • This flag enables word-boundaries in the specpp source preprocessor.

    • -m literal.pm
    • specpp
    • -m literal\.pm\b

    • This flag selects the "literal" preprocessor modules for the specpp source preprocessor. This module allows literal strings to pass through the preprocessor without any macros in the string being replaced.

    • -I., -I./netcdf/include
    • -I\.(?:/netcdf/include)?(?:\s+|$)

    • This flag is included automatically by Spec/object.pm.

    • wordsizes
    • -D[IDRL]WORDSIZE=\d+\b

    • This flag sets the default sizes (in byte) for various data types. It is defined in object.pm and cannot be changed.

    • -DTRIEDNTRUE
    • [-/]DTRIEDNTRUE(?:=\S*)?\b

    • Default flag defined in object.pm and cannot be changed.

    • -DLIMIT_ARGS
    • [-/]DLIMIT_ARGS(?:=\S*)?\b

    • Default flag defined in object.pm and cannot be changed.

    • -DNMM_MAX_DIM
    • [-/]DNMM_MAX_DIM(?:=\S*)?\b

    • Default flag defined in object.pm and cannot be changed.

    • -DCONFIG_BUF_LEN
    • [-/]DCONFIG_BUF_LEN(?:=\S*)?\b

    • This macro defines the size of the unused configuration buffer in words. It is set in object.pm and cannot be changed.

    • -DMAX_DOMAINS_F
    • [-/]DMAX_DOMAINS_F(?:=\S*)?\b

    • This macro sets the maximum number of domains (number of grids the model will be running). It is set in object.pm and cannot be changed.

    • -DEM_CORE=1, -DNMM_CORE=0, -DCOAMPS_CORE=0, -DEXP_CORE=0
    • [-/]D(?:NMM|COAMPS|EM|EXP)_CORE=\d+

    • These macros determine whether or not the named model core is built. The default values set in object.pm may not be changed.

    • -DF90_STANDALONE
    • [-/]DF90_STANDALONE(?:=\S*)?\b

    • For standalone code. The default value set in object.pm may not be changed.

    • -DINTIO
    • [-/]DINTIO(?:=\S*)?\b

    • This flag specifies that the ext_internal I/O module should be used. The default value set in object.pm may not be changed.

    • -DNETCDF
    • [-/]DNETCDF(?:=\S*)?\b

    • The flag specifies the use of NetCDF for I/O. The default value set in object.pm may not be changed.

    • -DNO_NAMELIST_PRINT
    • [-/]DNO_NAMELIST_PRINT(?:=\S*)?\b

    • Does not output the namelist input if set. The default value set in object.pm may not be changed.

Note that many of the flags classified as "forbidden" are classified as such only because they were not needed when doing the initial porting work for CPU2006, and because the impact of their use was not evaluated.

    • -DALWAYS_NC_SHARE
    • [-/]DALWAYS_NC_SHARE(?:=\S*)?\b

    • This flag causes NetCDF to attempt to set NC_SHARE on files it opens. It is used for debugging, and changing its value from the default is forbidden.

    • -DANSI_C_preprocessor
    • [-/]DANSI_C_preprocessor(?:=\S*)?\b

    • This flag is used by cfortran.h to determine whether it can use preprocessor features defined by the ANSI C standard. Changing its default value is forbidden.

    • -DFLOATFUNCTIONTYPE=XXX, -DASSIGNFLOAT=XXX, -DRETURNFLOAT=XXX
    • [-/]D(?:FLOATFUNCTIONTYPE|ASSIGNFLOAT|RETURNFLOAT)(?:=\S*)?\b

    • This flag (if necessary) is defined by the compilation system. Changing its default value is forbidden.

    • -DAUTODOC_BUILD
    • [-/]DAUTODOC_BUILD(?:=\S*)?\b

    • This flag causes WRF to not exit immediately after writing its inital restart file. Changing its default value is forbidden.

    • -DHAVE_ALLOCA
    • [-/]DHAVE_ALLOCA(?:=\S*)?\b

    • This macro indicates that alloca is available as a macro or function. It is set by default; if you need for it to be un-set, use -DSPEC_CPU_NO_HAVE_ALLOCA.

    • -DHAVE_ALLOCA_H
    • [-/]DHAVE_ALLOCA_H(?:=\S*)?\b

    • This macro indicates that the alloca.h header file is available. It is set by default; if you need for it to be un-set, use -DSPEC_CPU_NO_HAVE_ALLOCA_H.

    • -DHAVE_ST_BLKSIZE
    • [-/]DHAVE_ST_BLKSIZE(?:=\S*)?\b

    • This macro indicates that "struct stat" has a member named "st_blksize". It is set by default; if you need for it to be un-set, use -DSPEC_CPU_NO_HAVE_ST_BLKSIZE.

    • -DCDEBUG
    • [-/]DCDEBUG(?:=\S*)?\b

    • This flag enables coordinate check debug output in NetCDF. It may not be used for a CPU2006 run.

    • -DCDF_ROUTINE_NAME
    • [-/]DCDF_ROUTINE_NAME(?:=\S*)?\b

    • This flag causes the variable cdf_routine_name to be set to the current function name in the NetCDF-2 bindings. This flag may not be used for CPU2006.

    • -DCFSUBASFUN
    • [-/]DCFSUBASFUN(?:=\S*)?\b

    • This flag causes cfortran.h to replace subroutines with function calls. It may not be used for CPU2006.

    • -DCOMMON_BLOCK
    • [-/]DCOMMON_BLOCK(?:=\S*)?\b

    • This macro defines a method to refer to Fortran common blocks from C functions. Its default value may not be changed.

    • -DCOPY_OUT
    • [-/]DCOPY_OUT(?:=\S*)?\b

    • This macro causes variables to be copied out of the grid data structure at various places in the WRF source code. It is used internally by WRF and may not be set explicitly.

    • -DCRAYFLOAT
    • [-/]DCRAYFLOAT(?:=\S*)?\b

    • This macro indicates that CRAY floating point (real*16) should be used. This may not be set for CPU2006.

    • -DD3VAR_IRY_KLUDGE
    • [-/]DD3VAR_IRY_KLUDGE(?:=\S*)?\b

    • This macro changes the order of some variables used for WRF I/O. It may not be set for CPU2006.

    • -DDEREF_KLUDGE
    • [-/]DDEREF_KLUDGE(?:=\S*)?\b

    • This macro enables a potentially dangerous workaround to keep certain Fortran 90 compilers from performing a copy-in/copy-out conversion on the grid data structure in some cases. See this page for more information.

      This flag may not be used for CPU2006.

    • -DDEST_RECORD=XXX, -DSOURCE_RECORD=XXX, -DSOURCE_REC_DEX=XXX
    • [-/]D(?:(?:DEST|SOURCE)_RECORD|SOURCE_REC_DEX)(?:=\S*)?\b

    • These macros are used to set the names of data structures used for assignment. They are used internally by WRF and may not be set explicitly.

    • -DDLL_EXPORT, -DDLL_NETCDF
    • [-/]DDLL_(?:EXPORT|NETCDF)\b

    • These macros are used when building parts of the WRF library as a DLL. They are not used for CPU2006 and may not be set.

    • -DDM_PARALLEL
    • [-/]DDM_PARALLEL(?:=\S*)?\b

    • This macro indicates that WRF should be built for parallel operation. This mode is not supported for CPU2006, and this macro may not be set.

    • -DDOUBLE_PRECISION
    • [-/]DDOUBLE_PRECISION(?:=\S*)?\b

    • This macro contains the name of the C type which can hold a double precision Fortran number. Its default value may not be changed.

    • -DDUMBCOPY
    • [-/]DDUMBCOPY(?:=\S*)?\b

    • If defined, causes slow copy routines to be used in interp_fcn. This may not be set for CPU2006.

    • -DENOERR=XXX, -DFLT_MAX=XXX, -DFLT_MAX_EXP=XXX, -DINT_MAX=XXX, -DLONG_MAX=XXX, -DO_BINARY=XXX, -DSEEK_SET=XXX, -DSHORT_MAX=XXX
    • [-/]D(?:ENOERR|FLT_MAX_EXP|(?:INT|FLT|LONG|SHORT)_MAX|O_BINARY|SEEK_SET)(?:=\S*)\b

    • These macros should all be defined by the compilation system. Setting them explicitly is forbidden.

    • -DIX_INT_MAX=XXX, -DIX_SHORT_MAX=XXX, -DSIZEOF_IX_INT=XXX, -DSIZEOF_IX_SHORT=XXX
    • [-/]D(?:IX_(?:INT|SHORT)_MAX|SIZEOF_IX_(?:INT|SHORT))(?:=\S*)?\b

    • These macros define sizes and ranges of the variable types used to hold XDR data. They are not used for CPU2006 and may not be changed.

    • -DFCALLSC_QUALIFIER
    • [-/]DFCALLSC_QUALIFIER(?:=\S*)?\b

    • This macro holds the qualifier (if any) needed for C functions called from Fortran. The default value may not be changed.

    • -DFORTRAN_REAL
    • [-/]DFORTRAN_REAL(?:=\S*)?\b

    • This macro contains the name of the C type which can hold a Fortran REAL number. Its default value may not be changed.

    • -DINL
    • [-/]DINL(?:=\S*)?\b

    • This is an internal WRF macro which indicates that inlining of some functions should take place. It may not be set or unset explicitly.

    • -DLN_TEST, -DINSTRUMENT, -DODEBUG
    • [-/]D(?:LN_TEST|INSTRUMENT|ODEBUG)\b

    • These flags enable some NetCDF debugging output. They may not be used for CPU2006.

    • -DINTERNAL_IO
    • [-/]DINTERNAL_IO(?:=\S*)?\b

    • This macro indicates that WRF's internal I/O routines should be used. Its default value may not be changed.

    • -DINTIO_NOTYET
    • [-/]DINTIO_NOTYET(?:=\S*)?\b

    • This macro indicates that WRF's internal I/O routines should be stubbed out. It may not be set for CPU2006.

    • -DLOCKNUMREC
    • [-/]DLOCKNUMREC(?:=\S*)?\b

    • This flag enables locking for NC.numrecs. It may not be used for CPU2006.

    • -DMAKE_PROGRAM
    • [-/]DMAKE_PROGRAM(?:=\S*)?\b

    • Creates a test program for testing function nc_inq_libvers(). This may not be used for CPU2006.

    • -DMAX_NC_OPEN
    • [-/]DMAX_NC_OPEN(?:=\S*)?\b

    • This macro defines the maximum number of files a process can have open. Its default value may not be changed.

    • -DMAX_PREPRO_ARGS
    • [-/]DMAX_PREPRO_ARGS(?:=\S*)?\b

    • This macro defines the maximum number of arguments that the C preprocessor can accept for macro definitions. Its default value may not be changed.

    • -DMM5_SINT
    • [-/]DMM5_SINT(?:=\S*)?\b

    • This macro is set unconditionally in the WRF source code and may not be set or unset explicitly.

    • -DNAMELIST_READ_UNIT=XXX, -DNAMELIST_READ_ERROR_LABEL=XXX
    • [-/]DNAMELIST_READ_(?:UNIT|ERROR_LABEL)(?:=\S*)?\b

    • These macros define constants used when reading the namelist. Their default values may not be changed.

    • -DNAN_VALUE
    • [-/]DNAN_VALUE(?:=\S*)?\b

    • This macro indicates that a value for NaN is available. Its default value may not be changed.

    • -DNC_ARRAY_GROWBY
    • [-/]DNC_ARRAY_GROWBY(?:=\S*)?\b

    • This macro holds the size (in elements) that arrays in NetCDF should be grown at one time. Its default value may not be changed.

    • -DNC_ENOMEM=XXX, -DNC_ERANGE=XXX
    • [-/]DNC_(?:ENOMEM|ERANGE)(?:=\S*)?\b

    • These macros contain the values of return codes used internally by NetCDF. Their default values may not be changed.

    • -DNF_DOUBLEPRECISION_IS_C_FLOAT=XXX, -DNF_INT1_IS_C_SIGNED_CHAR=XXX, -DNF_INT2_IS_C_SHORT=XXX, -DNF_INT_IS_C_INT=XXX, -DNF_REAL_IS_C_DOUBLE=XXX, -DNF_REAL_IS_C_FLOAT=XXX
    • [-/]DNF_(?:NF_DOUBLEPRECISION_IS_C_FLOAT|NF_INT1_IS_C_SIGNED_CHAR|NF_INT2_IS_C_SHORT|NF_INT_IS_C_INT|NF_REAL_IS_C_DOUBLE|NF_REAL_IS_C_FLOAT)(?:=\S*)\b

    • These macros are used by NetCDF to determine which native C types fit which native Fortran types. Their default values may not be changed directly.

    • -DNO_IEEE_FLOAT
    • [-/]DNO_IEEE_FLOAT(?:=\S*)?\b
