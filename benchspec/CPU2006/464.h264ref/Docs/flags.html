464.h264avc Flags

These are flags that appear in the benchmark sources, categorized as to whether they are portability candidates, mandatory, or forbidden.

Sections

Selecting one of the following will take you directly to that section:

  • Portability Flags
  • Mandatory Flags
  • Forbidden Flags

Portability Flags

  • -DSPEC_CPU_AIX
  • -DSPEC_CPU_NO_INTTYPES
  • -DWIN32
  • -D__GNUC__
  • -DSPEC_CPU_NEED_MINMAX
    • -DSPEC_CPU_AIX
    • [-/]DSPEC_CPU_AIX(?:=\S*)?\b

    • This flag allow correct compilation and execution of 464.h264avc on the AIX platform.

    • -DSPEC_CPU_NO_INTTYPES
    • [-/]DSPEC_CPU_NO_INTTYPES(?:=\S*)?\b

    • This flag can be used as a portability flag on systems which do not have the standard header file "stdint.h". Inclusion of this flag defines the value of INT64_MIN (if it is not defined) and sets the "int64" type (if it is not set already)

    • -DWIN32
    • [-/]DWIN32(?:=\S*)?\b

    • This flag is set when building 464.h264avc on Windows systems.

    • -D__GNUC__
    • [-/]D__GNUC__(?:=\S*)?\b

    • This flag is defined at two locations. At both locations, it is used in conjunction with the WIN32 flag to distinguish non-Windows environment.

    • -DSPEC_CPU_NEED_MINMAX
    • [-/]DSPEC_CPU_NEED_MINMAX(?:=\S*)?\b

    • If your platform does not provide min() and max() functions (or macros), use this flag.

Mandatory Flags

  • -DSPEC_CPU
    • -DSPEC_CPU
    • [-/]DSPEC_CPU(?:=\S*)?\b

    • This preprocessor macro is always defined when compiling any benchmark in CPU2006. All platforms get the source lines enclosed within this macro block.

    • -DBEST_NZ_COEFF

    • Use of this flag is forbidden.

    • -DBI_PREDICTION
    • [-/]DBI_PREDICTION(?:=\S*)?\b

    • BI_PREDICTION is explicitly set in "defines.h". This flag is therefore not permitted on the command-line.

    • -DCONTEXT_INI_C
    • [-/]DCONTEXT_INI_C(?:=\S*)?\b

    • This flag is explicitly declared in "context_ini.c" file to indicate presence of CABAC context initialization. This flag is therefore not permitted on the command-line.

    • -DINCLUDED_BY_CONFIGFILE_C
    • [-/]DINCLUDED_BY_CONFIGFILE_C(?:=\S*)?\b

    • This flag is explicitly declared in "configfile.c" file to indicate that the parameters must be read from a config file. This flag is therefore not permitted on the command-line.

    • -DINT64_MIN
    • [-/]DINT64_MIN(?:=\S*)?\b

    • Defines the value of INT64_MIN if is not defined. This flag is protected by SPEC_CPU_NO_INTTYPES flag. C99 standards suggests that INT64_MIN be defined in "stdint.h". If you have defined flag SPEC_CPU_NO_INTTYPES and no definition is found for INT64_MIN, the program explicitly assigns value to INT64_MIN to (-9223372036854775807LL - 1LL).

    • -DKS_MV
    • [-/]DKS_MV(?:=\S*)?\b

    • This flag is explicitly declared in "rdopt.c" file. This flag is therefore not permitted on the command-line.

    • -DPMVFAST
    • [-/]DPMVFAST(?:=\S*)?\b

    • Sets variable "mvblocktype" which is actually NOT used in the program. Setting this flag on the command-line during compile is therefore forbidden.

    • -DPRINT_FMO_MAPS
    • [-/]DPRINT_FMO_MAPS(?:=\S*)?\b

    • Prints FMO maps to stdout. Setting this flag will generate invalid output resulting in verification error. Setting this flag on the command-line during compile is therefore forbidden.
