465.tonto Flags

These are flags that appear in the benchmark sources, categorized as to whether they are portability candidates, mandatory, or forbidden.

Portability Flags

  • -DBROKEN_PURE
  • -DBROKEN_TYPE_INITIALIZATION
  • -DLANGUAGE_FORTRAN_90
  • -DNOGENERIC
    • -DBROKEN_PURE
    • [-/]DBROKEN_PURE(?:=\S*)?\b

    • This flag disables several keywords having to do with "pure" routines. It is implied by LANGUAGE_FORTRAN_90, which would presumably be the more common way for it to get set, rather than appearing by itself on a command line.

    • -DBROKEN_TYPE_INITIALIZATION
    • [-/]DBROKEN_TYPE_INITIALIZATION(?:=\S*)?\b

    • This flag disables some setting of initial values. It is implied by LANGUAGE_FORTRAN_90, which would presumably be the more common way for it to get set, rather than appearing by itself on a command line.

    • -DLANGUAGE_FORTRAN_90
    • [-/]DLANGUAGE_FORTRAN_90(?:=\S*)?\b

    • Some Fortran compilers have some limitations; setting LANGUAGE_FORTRAN_90 will set BROKEN_PURE and BROKEN_TYPE_INITIALIZATION

    • -DNOGENERIC
    • [-/]DNOGENERIC(?:=\S*)?\b

    • NOGENERIC appears to cater for a Fortran compiler that has difficulties with Fortran generic function references. Although it is conceivable that a contemporary compiler might need this flag, its use should be questioned.

Mandatory Flags

  • -w
  • -m literal.pm, -m tonto.pm
  • -DSPEC_CPU
  • -DUSE_ERROR_MANAGEMENT
  • -DUSE_PRE_AND_POST_CONDITIONS
    • -w
    • specpp
    • -w\b

    • This flag enables word-boundaries in the specpp source preprocessor.

    • -m literal.pm, -m tonto.pm
    • specpp
    • -m (?:literal|tonto)\.pm\b

    • These flags select preprocessor modules for the specpp source preprocessor.

    • -DSPEC_CPU
    • [-/]DSPEC_CPU(?:=\S*)?\b

    • The SPEC version of the program omits some output, and also has a different convergence constant in scf. The flag SPEC_CPU is mandatory when running the benchmark version.

    • -DUSE_ERROR_MANAGEMENT
    • [-/]DUSE_ERROR_MANAGEMENT(?:=\S*)?\b

    • This flag is set automatically by Spec/object.pm.

    • -DUSE_PRE_AND_POST_CONDITIONS
    • [-/]DUSE_PRE_AND_POST_CONDITIONS(?:=\S*)?\b

    • This flag is set automatically by Spec/object.pm.

Forbidden Flags

  • -DESSL
  • -DF90_UNIX_IO
  • -DFLUSH
  • -DLITERAL_REVERSE
  • -DMPI
  • -DSERVICE_ROUTINES
  • USPEC_CPU
  • -DUSE_CALL_STACK_MANAGEMENT
  • -DUSE_PROFILING
    • -DESSL
    • [-/]DESSL(?:=\S*)?\b

    • This flag would cause tonto to use ESSL instead of LAPACK; the SPEC benchmark uses LAPACK.

    • -DF90_UNIX_IO
    • [-/]DF90_UNIX_IO(?:=\S*)?\b

    • It appears that this flag would cause a reference to a part of tonto that is not included in the SPEC benchmark, so it is therefore forbidden.

    • -DFLUSH
    • [-/]DFLUSH(?:=\S*)?\b

    • This flag might be useful in debugging, but is forbidden in a SPEC run.

    • -DLITERAL_REVERSE
    • [-/]DLITERAL_REVERSE(?:=\S*)?\b

    • LITERAL_REVERSE would affect the operation of the preprocessor itself; its effect is untested but quite likely its use would be an annoying faux pas.

    • -DMPI
    • [-/]DMPI(?:=\S*)?\b

    • The MPI support in of tonto is not used in SPEC CPU (though it might potentially be of interest in some other SPEC benchmarks).

    • -DSERVICE_ROUTINES
    • [-/]DSERVICE_ROUTINES(?:=\S*)?\b

    • It appears that this flag would cause a reference to a part of tonto that is not included in the SPEC benchmark, so it is therefore forbidden.

    • USPEC_CPU
    • [-/]USPEC_CPU(?:=\S*)?\b

    • The SPEC version of the program omits some output, and also has a different convergence constant in scf. The flag SPEC_CPU is mandatory when running the benchmark version.

    • -DUSE_CALL_STACK_MANAGEMENT
    • [-/]DUSE_CALL_STACK_MANAGEMENT(?:=\S*)?\b

    • Would be used only in debugging; forbidden in a SPEC run.

    • -DUSE_PROFILING
    • [-/]DUSE_PROFILING(?:=\S*)?\b

    • This flag is part of tonto's own profiling mechanisms. It could be used outside the tools harness if one were studying the code, but its use iwithin an actual SPEC benchmark run is forbidden.
