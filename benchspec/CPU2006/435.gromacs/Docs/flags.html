435.gromacs Flags

These are flags that appear in the benchmark sources, categorized as to whether they are portability candidates, mandatory, or forbidden.

Selecting one of the following will take you directly to that section:

Portability Flags

  • -DSPEC_CPU_APPEND_UNDERSCORE
    • -DSPEC_CPU_APPEND_UNDERSCORE
    • [-/]DSPEC_CPU_APPEND_UNDERSCORE(?:=\S*)?\b

    • For compilers that export symbols in uppercase with an underscore.

Mandatory Flags

  • -I.
  • -DHAVE_CONFIG_H
  • -DSPEC_CPU
  • -DSPEC_CPU_WINDOWS
    • -I.
    • -I\.(?:\s+|$)

    • This flag is included automatically by Spec/object.pm.

    • -DHAVE_CONFIG_H
    • [-/]DHAVE_CONFIG_H(?:=\S*)?\b

    • This should always be defined when running gromacs, since it means the configuration file is present. When including Gromacs headers as a library from external code it is not defined.

    • -DSPEC_CPU
    • [-/]DSPEC_CPU(?:=\S*)?\b

    • -DSPEC_CPU_WINDOWS
    • [-/]DSPEC_CPU_WINDOWS(?:=\S*)?\b

    • Portabilty for windows

  • -Drewind
    • -DALLOW_OFFDIAG_LT_HALFDIAG
    • [-/]DALLOW_OFFDIAG_LT_HALFDIAG(?:=\S*)?\b

    • This is just an development/debug flag for testing degenerate cases. All routines in Gromacs can work with general triclinic simulation cells instead of only rectangular ones. In normal simulations skewed cells are always restored, but this #define can be used to allow degenerate cells for testing. In that case we need to include some extra stuff in neighborsearching, which makes the code somewhat slower. It is NEVER used for production runs.

    • -DBUFSIZE
    • [-/]DBUFSIZE(?:=\S*)?\b

    • Default length of some line input buffers (char arrays) when reading from text files.

    • -DCHECK_MPI_ENV
    • [-/]DCHECK_MPI_ENV(?:=\S*)?\b

    • On some hosts a particular environment variable (e.g. LAM_WORLD) will be set if an MPI environment/daemon is started. If this define is set to a variable name a check is done to start a parallel run. It is NOT enabled by default.

    • -DCPLUSPLUS
    • [-/]DCPLUSPLUS(?:=\S*)?\b

    • Old, non-standard flag for detecting C++ compilers when Gromacs is called as a library from C++ code. We're gradually moving to the ANSI __cplusplus instead.

    • -DCYLDEBUG
    • [-/]DCYLDEBUG(?:=\S*)?\b

    • Debug flag for cylindrical pulling. Never used by default.

    • -DDEBSW
    • [-/]DDEBSW(?:=\S*)?\b

    • Flag to enable separate debugging of the tabulated switched nonbonded interactions. Never used by default.

    • -DDEBUG
    • [-/]DDEBUG(?:=\S*)?\b

    • Define to get lots of debug messages on stdout and stdlog. Never used by default.

    • -DDEBUG2
    • [-/]DDEBUG2(?:=\S*)?\b

    • Even more (=everything) debug messages. Never used by default.

    • -DDEBUGEBIN
    • [-/]DDEBUGEBIN(?:=\S*)?\b

    • Debug flag for our energy bin code. Not enabled by default.

    • -DDEBUGGCT
    • [-/]DDEBUGGCT(?:=\S*)?\b

    • Debug flag for "General Coupling Theory". Never used by default.

    • -DDEBUGIDEF
    • [-/]DDEBUGIDEF(?:=\S*)?\b

    • Debug flag for interaction definition blocks. Never used by default.

    • -DDEBUGMSHIFT
    • [-/]DDEBUGMSHIFT(?:=\S*)?\b

    • Debug flag for molecular shift code. Never used by default.

    • -DDEBUGPAR
    • [-/]DDEBUGPAR(?:=\S*)?\b

    • Debug flag for general parallel send/receive calls. Never used by default.

    • -DDEBUGRBIN
    • [-/]DDEBUGRBIN(?:=\S*)?\b

    • Debug flag to create a small test program for checking parallel data aggregation. Never used by default.

    • -DDEBUGSHAKE
    • [-/]DDEBUGSHAKE(?:=\S*)?\b

    • Debug flag for SHAKE, which is one of the bond constraint algorithms. Never enabled by default.

    • -DDEBUG_GMX
    • [-/]DDEBUG_GMX(?:=\S*)?\b

    • By default the debug_gmx() function does nothing. When this flag is set, it writes the current node id, file, and line number to the standard log (if available), or alternatively stderr. Never used by default.

    • -DDEBUG_NRNB
    • [-/]DDEBUG_NRNB(?:=\S*)?\b

    • Flag to debug the flop accounting (just used for statistics). Never used by default.

    • -DDEBUG_PBC
    • [-/]DDEBUG_PBC(?:=\S*)?\b

    • Debug flag for the periodic boundary condition code. Never enabled by default.

    • -DDEBUG_PRES
    • [-/]DDEBUG_PRES(?:=\S*)?\b

    • Debug flag for pressure calculation code. Never enabled by default.

    • -DDISABLE_WATERWATER_LOOPS
    • [-/]DDISABLE_WATERWATER_LOOPS(?:=\S*)?\b

    • This flag disables the water-water optimized inner loops. Nowadays it is just for debugging.

    • -DDOUBLE
    • [-/]DDOUBLE(?:=\S*)?\b

    • Defined when compiling a double precision Gromacs version.

    • -DFALSE
    • [-/]DFALSE(?:=\S*)?\b

    • if FALSE is defined we use it as a boolean value, otherwise we define it to 0.

    • -DFFTW_NAME_FFTW
    • [-/]DFFTW_NAME_FFTW(?:=\S*)?\b

    • Precision prefix (d,s, etc) when using the FFTW library calls. The SPEC version is compiled without FFTW.

    • -DFFT_WORKSPACE
    • [-/]DFFT_WORKSPACE(?:=\S*)?\b

    • Whether to use extra workspace for FFTs. This is enabled by default, but since the SPEC version is compile without FFTW it does not have any effect.

    • -DGROMACS_MAGIC
    • [-/]DGROMACS_MAGIC(?:=\S*)?\b

    • Magic number written at the start of frames in binary files, so we can identify them as Gromacs files.

    • -DHACK
    • [-/]DHACK(?:=\S*)?\b

    • Embarrassing debug hack to test part of the PPPM algorithm. Not used anymore.

    • -DHAVE_BOOL
    • [-/]DHAVE_BOOL(?:=\S*)?\b

    • Defined if the system has a "bool" type (otherwise we define it).

    • -DHAVE_FSEEKO
    • [-/]DHAVE_FSEEKO(?:=\S*)?\b

    • Defined if the system has 64-bit file access.

    • -DHAVE_LIBMASSV_ANY
    • [-/]DHAVE_LIBMASSV_ANY(?:=\S*)?\b

    • Defined if the IBM MASS libraries are present. This code is disabled in the SPEC version.

    • -DHAVE_MOTIF
    • [-/]DHAVE_MOTIF(?:=\S*)?\b

    • Defined if the Motif library is present, so we can do some graphical stuff. This code is disabled in the SPEC version.

    • -DMPI_TEST
    • [-/]DMPI_TEST(?:=\S*)?\b

    • Debug flag for MPI, introduce extra checking for each call. This part is disabled in the SPEC code.

    • -DM_2PI
    • [-/]DM_2PI(?:=\S*)?\b

    • -DM_PI
    • [-/]DM_PI(?:=\S*)?\b

    • -DNO_CLOCK
    • [-/]DNO_CLOCK(?:=\S*)?\b

    • Defined if no clock routines are available. Just used for statistics anyway, and the output is disabled in the SPEC code.

    • -DNO_GETPID
    • [-/]DNO_GETPID(?:=\S*)?\b

    • Defined if it is not possible to get a process ID from the system. This is just used as an optional way to get a real 'random' random seed. This part of the code is not used in the SPEC test.

    • -DNO_PIPE
    • [-/]DNO_PIPE(?:=\S*)?\b

    • Defined if no pipes are available on this system. Only used to compress/uncompress IO on-the-fly, not enabled in the SPEC test.

    • -DNO_PWUID
    • [-/]DNO_PWUID(?:=\S*)?\b

    • Defined if pwd.h is not present, so we cannot get name/uid of the user. This is only used to add a comment at the top of a log file, which anyway is disabled in the SPEC code.

    • -DNS5DB
    • [-/]DNS5DB(?:=\S*)?\b

    • Debug flag for grid-based neighborsearching.

    • -DPRAGMAS
    • [-/]DPRAGMAS(?:=\S*)?\b

    • Add a cray pragma to ignore vector dependencies in the next loop if set. Not enabled by default.

    • -DSAFE
    • [-/]DSAFE(?:=\S*)?\b

    • Used to check and work around an old compiler bug. No longer used by default.

    • -DSEGV
    • [-/]DSEGV(?:=\S*)?\b

    • -DSOFTWARE_RECIP
    • [-/]DSOFTWARE_RECIP(?:=\S*)?\b

    • -DUSE_THREADS

    • -DUSE_VECTOR

    • -D__sgi
